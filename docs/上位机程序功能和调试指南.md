## 3. 串口参数

当前临时测试固件和上位机默认使用：

- 串口：通常为 `COM11`
- 波特率：`115200`
- 数据位：8
- 校验位：无
- 停止位：1
- 流控：无

注意：之前工程中曾使用 `921600`，但为了让 ST-LINK Virtual COM Port 双向通信更稳定，当前临时测试阶段已统一改为 `115200`。

## 4. 图形界面功能

### 4.1 连接区域

界面顶部是串口连接区：

- `串口`：选择或手动输入串口号。
- `刷新`：重新扫描电脑当前可用串口。
- `波特率`：默认 `115200`。
- `连接/断开`：打开或关闭串口。

推荐操作顺序：

1. 点击 `刷新`。
2. 选择 `STMicroelectronics STLink Virtual COM Port` 对应的串口，当前通常是 `COM11`。
3. 确认波特率为 `115200`。
4. 点击 `连接`。

连接成功后，状态栏会显示已连接信息。如果接收器固件正在运行临时测试模式，界面会收到 `PC_ALIVE` 心跳，并显示接收器在线。

### 4.2 扫描参数

扫描参数包括：

- `起点`：DAC 起始码值，范围 `0` 到 `65535`。
- `终点`：DAC 结束码值，范围 `0` 到 `65535`。
- `点数`：扫描点数，当前建议使用 `100`。
- `等待(ms)`：每个点的等待时间，当前临时测试固件不会真实等待传感器稳定，只用于协议验证。
- `平均`：平均次数，当前临时测试固件只记录该参数，不执行真实平均。

点击 `开始扫描` 后，上位机会发送 `CMD_SCAN_START` 命令。

当前临时接收器固件会模拟返回：

1. `EVT_ACK`
2. `EVT_SCAN_BEGIN`
3. 多个 `EVT_SCAN_POINTS`
4. `EVT_SCAN_END`

扫描点会显示在左侧点表中，进度条会随收到的点数更新。

### 4.3 DAC 设置

`DAC` 输入框用于填写目标 DAC 码值。

点击 `设置 DAC 并保存` 后，上位机会发送 `CMD_SET_DAC` 命令。

当前临时接收器固件会：

- 返回 `EVT_ACK`
- 更新模拟 DAC 值
- 翻转绿色 LED，用于辅助观察命令是否到达

注意：当前只是接收器本地模拟保存行为，并没有真正写入传感器 DAC。真实掉电保存需要后续传感器端 `BLE_Custom` 支持写命令和非易失存储后实现。

### 4.4 终止扫描

点击 `终止` 后，上位机会发送 `CMD_ABORT` 命令。

当前临时接收器固件会：

- 返回 `EVT_ACK`
- 返回一个 `EVT_SCAN_END`

这样 GUI 可以把当前扫描流程收束到结束状态。

### 4.5 通信日志

右侧日志区显示：

- 发送命令，例如 `TX CMD_SCAN_START`
- 接收事件，例如 `RX EVT_ACK`
- 协议解析后的扫描点信息
- 错误提示
- 接收器心跳 `PC_ALIVE`

GUI 对 `PC_ALIVE` 做了节流处理，不会每秒刷满日志。

### 4.6 扫描点表

左侧表格显示三列：

- `序号`
- `DAC`
- `Lock-in`

当前临时固件生成的是模拟 `Lock-in` 数据，用于验证显示、分包、导出和流程控制。

### 4.7 导出点表

点击 `导出点表` 可以将当前扫描点保存为 CSV 文件。

导出格式：

```csv
index,dac,lockin
0,0,1000
1,662,4093
...
```

CSV 使用 `utf-8-sig` 编码，方便 Excel 直接打开。

### 4.8 协议自测

点击 `协议自测` 会在本机内部测试协议打包和拆包，不依赖串口和单片机。

如果协议自测失败，说明上位机程序自身的协议解析存在问题；如果协议自测成功但串口无响应，则应优先排查串口、波特率、固件烧录和硬件连接。

## 5. 当前二进制协议概览

当前上位机和接收器之间使用二进制帧。

帧结构：

```text
Magic      2 bytes   A5 5A
Version    1 byte    当前为 01
Type       1 byte    命令或事件类型
Flags      1 byte    标志位
ScanId     2 bytes   小端
Seq        2 bytes   小端
Length     1 byte    payload 长度
Payload    N bytes
CRC16      2 bytes   小端，覆盖 Version 到 Payload
```

主要命令：

| 类型 | 名称 | 方向 | 说明 |
| --- | --- | --- | --- |
| `0x10` | `CMD_SCAN_START` | PC 到接收器 | 开始扫描 |
| `0x11` | `CMD_SET_DAC` | PC 到接收器 | 设置 DAC，当前 flags bit0 表示保存 |
| `0x12` | `CMD_ABORT` | PC 到接收器 | 终止扫描 |

主要事件：

| 类型 | 名称 | 方向 | 说明 |
| --- | --- | --- | --- |
| `0x80` | `EVT_ACK` | 接收器到 PC | 命令确认 |
| `0x81` | `EVT_NACK` | 接收器到 PC | 命令拒绝或错误 |
| `0x90` | `EVT_SCAN_BEGIN` | 接收器到 PC | 扫描开始 |
| `0x91` | `EVT_SCAN_POINTS` | 接收器到 PC | 扫描点数据 |
| `0x92` | `EVT_SCAN_END` | 接收器到 PC | 扫描结束 |
| `0xA0` | `EVT_BLE_RAW` | 接收器到 PC | BLE 原始数据或临时心跳 |

当前 `EVT_SCAN_POINTS` payload 格式：

```text
count      1 byte
point[0]   5 bytes: index u8, dac u16 little-endian, lockin u16 little-endian
point[1]   5 bytes
...
```

当前临时固件每包发送 10 个点。

## 6. 命令行调试方法

虽然主要使用 GUI，但命令行工具适合快速定位问题。

### 6.1 协议自测

```powershell
D:\anaconda\envs\dielectric_env\python.exe D:\stm32project\BLE_vscode\tools\dac_scan_host.py --self-test
```

正常输出应包含：


```powershell

```

如果接收器固件正常运行，应该能看到：

### 6.3 发送扫描命令

```powershell

```

正常情况下应看到：

```text

rx EVT_ACK

rx EVT_SCAN_POINTS
rx EVT_SCAN_END
```


正常情况下应看到：


### 6.5 终止扫描

```text
rx EVT_ACK: target=CMD_ABORT status=OK
rx EVT_SCAN_END
```

## 7. 接收器临时测试固件说明

当前接收器工程中加入了临时测试模式：

```c
#define PC_TEST_MODE 1U
```

位置：

```text
D:\stm32project\BLE_vscode\BLE_RECEIVER\STM32_WPAN\App\p2p_client_app.c
```

该模式的作用是：

- 不依赖传感器端 BLE 写特征。
- 接收器直接响应上位机命令。
- 模拟 DAC 扫描点。
- 方便测试 GUI 的所有按钮、表格、日志、导出和协议解析。

当前还在 `main.c` 中加入了 `PC_ALIVE` 心跳，用于确认接收器 TX 到 PC 的串口方向正常。

后续接入真实传感器后，应移除或关闭这些临时逻辑：

1. 将 `PC_TEST_MODE` 改为 `0U` 或删除临时代码。
2. 移除 `main.c` 中的 `PC_ALIVE` 心跳。
3. 将 `CMD_SCAN_START / CMD_SET_DAC / CMD_ABORT` 转发到传感器端 BLE 写特征。
4. 将传感器返回的扫描数据转换为正式 `EVT_SCAN_*` 事件。

## 8. 常见问题排查

### 8.1 点击运行后没有 GUI

检查 VSCode 当前运行的是不是：

```text
D:\stm32project\BLE_vscode\tools\dac_scan_host.py
```

或直接运行：

```powershell
D:\anaconda\envs\dielectric_env\python.exe D:\stm32project\BLE_vscode\tools\dac_scan_gui.py
```

### 8.2 提示找不到 serial 或 pyserial

说明当前 Python 环境没有安装 `pyserial`。

安装：

```powershell
D:\anaconda\envs\dielectric_env\python.exe -m pip install pyserial
```

验证：

```powershell
D:\anaconda\envs\dielectric_env\python.exe -c "import serial; print(serial.VERSION)"
```

### 8.3 串口打不开

可能原因：

- COM 口选错。
- 串口已被另一个程序占用。
- 之前的命令行监听程序还没关闭。
- STM32CubeProgrammer 或调试器正在占用相关资源。

处理方法：

1. 关闭其它串口助手、命令行监听程序和 GUI。
2. 点击 GUI 的 `刷新`。
3. 重新选择 `STMicroelectronics STLink Virtual COM Port`。
4. 再次连接。

### 8.4 能连接但没有任何数据

先用命令行监听：

```powershell
D:\anaconda\envs\dielectric_env\python.exe D:\stm32project\BLE_vscode\tools\dac_scan_host.py --port COM11 --timeout 5
```

如果没有 `PC_ALIVE`：

- 确认固件是否已烧录最新版。
- 确认波特率是否为 `115200`。
- 确认选择的是 ST-LINK VCP 对应串口。
- 按一下开发板 Reset 后再监听。

### 8.5 能收到 PC_ALIVE，但发送扫描无响应

说明接收器到 PC 的 TX 方向正常，但 PC 到接收器的 RX 方向可能异常。

检查：

- GUI 波特率是否为 `115200`。
- 命令是否真的发送，日志中应有 `TX CMD_SCAN_START`。
- 是否选中了正确 COM 口。
- 是否有其它程序同时打开串口。

当前已验证 `COM11 @ 115200` 可以完成双向通信。

### 8.6 扫描点显示不完整

当前 100 点扫描会分 10 包返回，每包 10 点。

如果显示点数不足：

- 检查日志中是否出现 `EVT_SCAN_END`。
- 检查是否中途点击了 `终止`。
- 增大命令行测试的 `--timeout`。
- GUI 模式下等待进度条到 100%。

### 8.7 CSV 导出乱码

当前导出使用 `utf-8-sig`，Excel 一般可以正常识别。

如果仍然乱码，可使用 Excel 的“数据 -> 从文本/CSV”导入，并选择 UTF-8 编码。

## 10. 当前阶段限制

当前已打通的是：

```text
上位机 GUI <-> ST-LINK VCP 串口 <-> 接收器本机临时模拟逻辑
```

当前尚未打通的是：

```text
上位机 GUI <-> 接收器 <-> BLE <-> 传感器端 DAC/扫描逻辑
```

因此，当前 GUI 中看到的扫描点是接收器模拟生成的数据，不是真实传感器数据。

后续开发重点：

1. 在 `BLE_Custom` 传感器端增加 BLE 写特征。
2. 定义传感器端命令解析：扫描开始、DAC 设置、终止。
3. 实现 DAC 掉电保存。
4. 实现真实扫描采样和平均。
5. 将传感器扫描结果通过 BLE Notify 发给接收器。
6. 接收器把真实结果转换为上位机协议事件。
